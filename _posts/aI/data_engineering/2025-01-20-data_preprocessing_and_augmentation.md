## 학습 안정화

### Label Smoothing
  - 설명: one-hot encoding된 label을 약간 부드럽게 만들어 모델이 overconfident하지 않도록 하는 기법  
  - 방법: 정답 클래스를 1이 아닌 0.9와 같이 조금 낮은 값을 주고, 나머지 0 값들에게 0.1을 나눠준다.  
  - 장점
    + 일반화 성능 향상
    + 학습 안정성 향상
  - 단점
    + 클래스 간의 관계를 반영 못함

### Stochastic Depth
  - 설명: 학습 중 일부 레이어를 무작위로 비활성화하여 네트워크 깊이를 줄이는 기법
  - 방법: 각 레이어에 대해 확률적으로 호라성화 여부 결정
  - 장점:
    + Gradient Vanishing 완화
    + 학습 속도 개선, 메모리 사용량 감소
    + Overfitting 완화
  - 단점:
    + 추론 단계에서 학습 단계와의 불일치 발생 가능


## Data Preprocessing

### Handling Missing Values (결측값 처리)
  - 설명: 데이터셋에서 누락된 값을 처리하는 방법
  - 방법:
    + 삭제
    + 대체: 중앙값, 평균, 최빈값 등으로 대체
  - 장점:
    + 누락 데이터가 많지 않을 경우 데이터를 깨끗하게 유지할 수 있다.
    + 대체 방법은 데이터 손실을 최소화한다.
  - 단점:
    + 손실이 많으면 삭제는 데이터 품질을 저하시킨다.
    + 부정확한 대체는 학습에 좋지 않다.

### Outlier Handling (이상치 처리)
  - 설명: 데이터에서 비정상적으로 벗어난 값을 처리
  - 방법:
    + 삭제: IQR 또는 Z-score 사용하여 이상치 식별 후 제거
    + 변환: 로그 변환 또는 범위 압축
  - 장점:
    + 극단적인 데이터가 모델에 미치는 영향을 줄인다.
  - 단점:
    + 비정상적인 데이터를 삭제하면 중요한 패턴을 잃을 수 있다.

### Normalization (데이터 정규화)
  - 설명: 데이터를 특정 범위로 변환하거나, 특정 분포로 스케일링
  - 방법:
    + MinMaxScaling: 특정 범위(ex: [0:1])로 변환
  - 장점:
    + 학습 안정성 향상
  - 단점:
    + MinMax는 이상치에 민감
    + 데이터의 원래 분포를 왜곡할 가능성이 있다

### Standardization (데이터 표준화)
  - 설명: 데이터를 평균이 0, 표준편차가 1인 분포로 변환
  - 방법: 정규화를 통해 변환
  - 장점:
    + 다양한 크기의 데이터를 비교 가능
  - 단점:
    + 정규 분포가 아닌 데이터에서는 효과가 제한적

### Categorical Encoding (카테고리형 데이터 인코딩)
  - 설명: 범주형 데이터를 수치형 데이터로 변환
  - 방법: 
    + One-Hot Encoding
    + Label Encoding
  - 장점:
    + 수치형 변환으로 머신러닝 모델에 적합
  - 단점:
    + One-Hot Encoding은 많은 범주에서는 차원이 폭발적으로 증가한다.
    + 레이블 인코딩은 순서 정보가 왜곡될 가능성이 있다.

### Data Sampling (데이터 샘플링)
  - 설명: 데이터 크기를 줄이거나 균형 있게 만들기 위한 샘플링
  - 방법: 
    + Under Sampling: 다수 클래스의 데이터를 줄임
    + Over Sampling: 소수 클래스의 데이터를 복제 또는 생성(SMOTE)
  - 장점:
    + 데이터 불균형 문제를 해결
  - 단점:
    + Under Sampling은 데이터 손실 위험
    + Over Sampling은 Overfitting 위험

### Log Transformation (로그 변환)
  - 설명: 데이터 스케일을 줄이기 위해 로그 함수 적용
  - 방법: $ y = \log{x} $
  - 장점:
    + 비대칭 데이터의 분표를 정규 분포에 가깝게 변환
  - 단점:
    + 값이 0이거나 음수인 데이터에는 사용 불가능

### Discretization (데이터 이산화)
  - 설명: 연속형 데이터를 이산형(범주형)으로 변환
  - 방법: 
    + Equal-Width: 같은 크기로 구간 분할
    + Equal-Frequency: 같은 빈도로 구간 분할
  - 장점:
    + 연속형 데이터의 복잡성을 줄이며, 범주형으로 변환 가능
  - 단점:
    + 구간 경계 설정에 따라 정보 손실 가능

### Text Cleaning (텍스트 정리)
  - 설명: 텍스트 데이터를 정리하고 불필요한 부분 제거
  - 방법:
    + 소문자 변환
    + 불용어(stop-words) 제거
    + 표제어(lemma) 추출
    + 어간(stemming) 추출
  - 장점:
    + 노이즈 제거
  - 단점:
    + 과도한 정리는 유용한 정보 제거 가능성 있다

### PCA (주성분 분석, Principal Component Analysis)
  - 설명: 고차원 데이터를 저차원으로 축소하면서 정보 손실 최소화
  - 방법: 데이터의 분산을 기준으로 가장 중요한 축을 선택
  - 장점:
    + 차원을 줄여 계산 비용 감소
    + 노이즈 제거 가능
  - 단점:
    + 데이터의 해석력이 떨어질 수 있다


## Data Augmentation

### RandAugment
  - 설명: 이미지에 다양한 변화(회전, 크기 조정, 밝기 변경, 잘림, 색상 변화, 대비 조절 등을 **무작위**로 적용)하는 기법
  - 방법:
    + 변환 함수 집합에서 N개의 증강 연산을 무작위로 선택 후 각 변환 함수에 대해 Magnitude(강도)를 무작위로 설정하여 이미지에 순차적으로 적용
  - 장점:
    + 데이터 다양성 증가: 일반화
    + overfitting 방지
    + 데이터셋이나 모델에 관계없이 적용 가능
  - 단점:
    + 무작위 증강으로 원본 데이터의 특징이 크게 변형될 수 있다.

### Random Erasing
  - 설명: 이미지 일부 영억을 지우거나 삭제하는 증강 기법
  - 방법:
    + 이미지에서 무작위 위치를 선정, 선택된 위치를 마스킹
  - 장점:
    + robust 학습: 안정성
    + 데이터 다양성 증가
  - 단점:
    + 중요한 특징이 지워질 수 있다.

### Mixup
  - 설명: 두개의 이미지를 선형적으로 혼합, 라벨도 동일한 비율로 혼합
  - 방법: 
    + 두개의 이미지를 흐릿하게 합성하는 것이다. 라벨도 혼합된다.
    + $ x_{new} = \lambda x_1 + (1 - \lambda) x_2 $ 공식을 사용
  - 장점:
    + overfitting 방지
    + 일반화 성능 향상
  - 단점:
    + 일부 도메인에서는 의미 있는 혼합이 어려울 수 있다.

### CutMix
  - 설명: Mixup과 유사하지만, 한 이미지의 일부 영역을 잘라 다른 이미지에 붙여 넣는 방식
  - 방법: 일부 영역을 다른 이미지의 일부분으로 대체하는 것이다. 라벨도 같이 혼합된다.
  - 장점:
    + 로컬 특징 학습: 특정 이미지 영역의 정보를 학습하도록 유도, Mixup 보다 로컬 학습 유리
    + 일반화 성능 향상
  - 단점:
    + 잘못된 영역 혼합 시 의미있는 정보 손실 가능
