## KNN (K-Nearest Neighbor)

 - K-최근접 이웃
 - 가장 가까운 K개의 이웃을 통해 결과 값을 추론
 - [**비모수**](/posts/AI_단어정리/#비모수)적 방식
 - [**인스턴스 기반 학습**](/posts/AI_단어정리/#인스턴스_기반_학습)
 - 훈련 데이터로 [**특징 벡터**](/posts/AI_단어정리/#특징_벡터) 시퀀스와 분류명 시퀀스(타겟 벡터 시퀀스)를 받음
 - [**하이퍼파라미터**](/posts/AI_단어정리/#하이퍼파라미터) : K (이웃 개수), 거리 측정 방법 (ex [**유클라디안 거리**](/posts/AI_단어정리/#유클라디안_거리), [**맨해튼 거리**](/posts/AI_단어정리/#맨해튼_거리), [**마할라노비스 거리**](/posts/AI_단어정리/#마할라노비스_거리)), 가중치
 - [**모델 파라미터**](/posts/AI_단어정리/#모델_파라미터) : 없음


## 기본적인 알고리즘

1. 훈련데이터 저장
2. 예측을 수행할 새로운 데이터 주입
3. 새로운 데이터에 대한 훈련 데이터간의 거리 측정
4. 거리가 가장 가까운 K개의 최근접 이웃을 찾음
5. 최근접 이웃을 바탕으로 새로운 데이터의 예측값 추론 (분류에서는 가반수, 회귀에서는 평균)

## 과대접합과 과소접합
- [**과대접합**](/posts/AI_단어정리/#과대접합) : k값을 증가시키면 일반화 되어 해결 가능
- [**과소접합**](/posts/AI_단어정리/#과소접합) : k값을 감소시키면 모델이 민감해 져 해결 가능

## 문제점과 해결방법

- 과반수 문제 : 과반수 의견에 따라, 빈번한 데이터가 예측값을 지배하는 경우 (편향된 예측)<br>
    -> 분류에 가중치 부여 <br>
    -> [**데이터 클러스터링**](/posts/AI_단어정리/#데이터_클러스터링) <br>
    -> 고차원 매핑 - 특징을 세분화 하여 개수를 늘리는 것 <br>

- K 값에 따른 예측 변화 : K의 값에 따라서 예측 값이 다름 <br>
    -> K 값을 [**휴리스틱 기법**](/posts/AI_단어정리/#휴리스틱_기법)으로 찾기

- 데이터의 크기가 오버헤드와 비례 : 데이터의 크기가 클수록 각 데이터와 새로운 데이터를 비교해야 하므로 처리 시간과 메모리 사용량이 증가하는 문제<br>
    -> 차원 축소 <br>

- 훈련 데이터 범위 밖의 데이터 예측 어려움 : 불가피한 문제
